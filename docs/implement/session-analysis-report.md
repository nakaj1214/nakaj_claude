## 1. スキルにすべきもの

繰り返し手動で行っているワークフローをスキル化する。

### 1-3. consistency-check（整合性チェック）
- **発生頻度**: 3回（trader で頻出）
- **現在の入力**: `他セッションでも機能を追加していたので整合性チェックして`
- **自動化内容**: 複数セッション並行開発後のファイル間矛盾・API不整合・型不一致を検出
- **期待効果**: 並行開発時のバグを事前防止

### 1-4. deploy-env-check（デプロイ環境事前チェック）

- **発生頻度**: 4回（trader で頻出）
- **現在の入力**: `Cloudflare Pages で動く？` `GitHub Actions で問題ない？`
- **自動化内容**: コードがデプロイ環境の制約（Node.js バージョン、API制限等）に適合するか検証
- **期待効果**: デプロイ失敗の事前防止
フォルダ名がtraderの時のみ

### 1-5. translate-docs（ドキュメント翻訳のスキル化）

- **発生頻度**: 2回
- **現在の方法**: `translate_to_japanese.py` をスクリプトとして手動実行
- **自動化内容**: 既存スクリプトをスキルとしてラップし、`/translate-docs` で呼び出し可能に
- **期待効果**: スクリプト実行の手間を省略

## 3. フックにすべきもの

イベント駆動で自動実行されるべき処理。

### 3-1. PreToolUse: deploy-config-reminder

- **イベント**: Bash ツールで `git push` または `deploy` コマンド実行前
- **処理**: プロジェクトの CLAUDE.md からデプロイ環境情報を読み取り、注意事項をリマインド
- **根拠**: trader で「Cloudflare で動く？」を毎回確認していた
- **例**: `このプロジェクトは Cloudflare Pages で動作します。Node.js 18+ が必要です`
フォルダ名がtraderの時のみ

### 3-2. PostToolUse: multi-session-conflict-detect

- **イベント**: Edit/Write ツール実行後
- **処理**: 直近の git log と diff を比較し、他セッションの変更と競合する可能性を検出
- **根拠**: trader で並行セッション後に整合性チェックが毎回必要だった
- **例**: `警告: src/api/index.ts は 2時間前に別セッションで変更されています`

### 3-3. PostToolUse: archive-on-delete

- **イベント**: ファイル削除（rm, 大量の Edit による内容消去）検出時
- **処理**: 削除対象を `archive/` フォルダにコピーしてからの削除を強制
- **根拠**: `削除はコメントアウトで` `archive フォルダに移動して` の指示が繰り返し発生
- **例**: `ファイルを削除する前に archive/ にバックアップしました`

## 4. ルールにすべきもの

常時または条件付きで注入されるコーディング・運用ルール。

### 4-1. 常時ロード: destructive-operations（破壊的操作の制限）

```markdown
# 破壊的操作の制限

- ファイル削除時は `archive/` フォルダに移動してから削除する
- コード削除はコメントアウトで行う（完全削除は後のセッションで確認後）
- `git reset --hard`, `git clean -f` は禁止
- データベースの DROP は禁止
```

- **根拠**: `破滅的操作を禁止し、プログラムを削除する際はコメントアウトで削除するように強制して` という明示的な指示

### 4-3. 条件付きロード: parallel-session-awareness（並行セッション注意）

```markdown
# 並行セッション注意

- 同一プロジェクトで複数セッションを使用している可能性がある
- 実装前に `git log --oneline -10` で直近の変更を確認する
- 大きな変更前に `git stash` で現在の作業を退避する
- 実装完了後に整合性チェック（型チェック、テスト実行）を行う
```

- **根拠**: `他セッションでも機能を追加していたので整合性チェックして` が繰り返し発生
