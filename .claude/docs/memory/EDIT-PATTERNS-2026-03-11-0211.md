## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `excel_range_select_copy.js` への30回超の編集 | UI 選択ハイライトという「視覚的に確認しないと判断できない」変更を、画面確認なしに仮説ベースで繰り返した | `verify-before-fix` スキルを必ず発動し、スクリーンショットで現状把握してから実装に進む |
| proposal.md が8回上書きされた | 要件が固まる前にファイルへ書き出し、対話的な修正をファイル上書きで行った | proposal は「確定した要件だけ書く」ルールを強化。修正は Edit で行い、Write(上書き)は最終版のみ |
| settings.json の matcher 往復 | 設定変更後の動作確認なし | 設定変更時は次リクエストで動作確認してから完了とする |
| Delete/Backspace が3回失敗 | フォーカス状態やイベントバブリングの調査が不十分なまま実装に進んだ | 失敗2回目の時点で `fix-escalation` スキルを発動し、視点を変えた調査を強制する |
| JS クラス追加後の CSS 対応漏れ | JS と CSS を別々に考えてしまい、ペア変更の意識が薄い | `js-css-sync-check` スキルを JS 編集後に毎回実行する（`PostToolUse` フックで自動促進） |
