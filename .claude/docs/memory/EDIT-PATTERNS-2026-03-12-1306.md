## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `plan.md` が1セッションで5回以上 Write される | Codex レビューで CHANGES_REQUIRED が繰り返され、計画全体を書き直している | `plan.md` は Edit で差分修正する。Write（全体書き直し）は最終手段に |
| `.claude/settings.json` の matcher が往復 | 変更後の動作確認なしに「うまくいかなかった」と判断してすぐ戻す | 変更前後でテストスクリプト実行を必須化（`test_hook.sh` 等）|
| `excel_range_select_copy.js` に対して3種類の異なるアプローチを試した | ブラウザでの動作確認なしに次のアプローチに移行している | `verify-before-fix` スキルを使い、antigravity IDE でのスクリーンショット確認を必須化 |
| `home.blade.php` の `daily_items_panel` 配置が4回変更 | HTML 構造の設計（どのコンテナに配置するか）を実装前に決めていない | proposal.md に「配置場所の図」を含めることを受入条件に追加 |
| `docs/implement/review.md` がセッション `4e0e035f` で12回生成 | Codex のレビューループが収束しない（毎回 CHANGES_REQUIRED）| proposal → plan 段階で Codex 可決基準を満たす粒度に落とし込む。`proposal-quality-gate` スキルのトリガーを強化 |
