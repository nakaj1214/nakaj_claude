## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `excel_range_select_copy.js` に3回のアプローチ変更（box-shadow → inset → ::after） | input 要素が td を覆う構造を事前に把握していなかった。設計前に DOM 構造確認が不足 | 実装前に `antigravity` でスクリーンショット取得し、DOM の重なり順を確認してから手法選択する |
| `plan.md` / `proposal.md` が8回上書きされた | ブラウザ確認なしに次の修正を書き続けた。UIの実際の見た目を確認してから要件書を書くべきだった | `verify-before-fix` スキルで「証拠収集 → 要件定義 → 実装」の順序を強制する |
| settings.json の PreToolUse matcher を変更後6分で revert | 変更の影響を事前に把握していなかった。動作確認が事後になった | settings.json 変更時は変更前に影響フック名を列挙してユーザーに確認する |
| Codex を Task サブエージェント経由で呼ぶとタイムアウト | ネスト実行（Claude → subagent → codex exec）で websocket/MCP 接続が二重になる | `codex-review` スキルを Bash 直接実行に修正済み（MEMORY.md と skills に反映済み） |
| JS でクラス名定義 → CSS にスタイル定義のペアが毎回発生 | JS/CSS が別ファイルに分離されているため、片方の更新を忘れやすい | `js-css-sync-check` スキルで自動チェック（作成済み） |
