## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| Codex 実行方式を Bash → MCP ツールに移行（MEMORY.md に記録済み） | `codex exec` の instruction 自動読み込みによるタイムアウト | `.mcp.json` 登録 + `base-instructions` パラメータ必須を rules に明記済み |
| `settings.json` の hook 設定変更後に動作未確認のまま進む → revert 多発 | 設定変更の即時検証手段がない | hook 設定変更時は `python3 -c "import json; json.load(open('.claude/settings.json'))"` で構文確認 + 次の Bash 実行で動作確認を必須化する |
| `plan-rewrite-alert` / `large-delete-guard` が staging に生成されたが有効化されていない | materialize 後の review-staged が未実施 | `/review-staged` を実行して staging の 3 候補を確認・有効化する |
| layout 試行錯誤（order.blade.php が 4 回変更） | HTML/CSS のレイアウト変更は動作確認なしに修正を重ねた | レイアウト変更前に `antigravity` でスクリーンショット取得 → verify-before-fix スキル適用 |
