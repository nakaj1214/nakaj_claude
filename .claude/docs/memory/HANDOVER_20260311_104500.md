## 発生した問題と解決策

### disabled inputでセル選択ができない

**問題:** disabled属性のinput要素がマウスイベントを親要素にバブルさせない
**解決策:** CSS `pointer-events: none` を `tbody td[data-row][data-col] input[disabled], input[readonly]` に適用
**注意点:** この変更により disabled/readonly input上でテキスト選択（手動コピー）はできなくなるが、ExcelRangeSelectCopy のコピー機能で代替される

## 注意点・落とし穴

- VSCode拡張のファイル変更通知（system-reminder）が毎メッセージに変更ファイル数分注入される → 1セッションで多数ファイルを変更するとトークン消費が加速する
- `home.blade.php` のdepartment_idチェックがユーザーにより手動削除されている（意図的）
- `edit-tracker` と `same-file-edit-warn` のPostToolUseフック出力もトークンを消費する
