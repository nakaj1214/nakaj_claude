## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `excel_range_select_copy.js` が繰り返し修正された | セル選択UIの要件が段階的に変化し、前の実装を前提としない設計が必要だった | 実装前に「outline描画の技術的制約（input が td 全体を覆うケース）」を先に確認するチェックリストを plan に含める |
| `plan.md` が同一セッション内で3〜4回書き直される | Codex レビューの指摘が計画全体に及ぶ場合、Edit では対応しきれない | plan.md は Edit で差分修正する前提で構造化。「書き直し判定」基準を設ける |
| `settings.json` の設定が行き戻りした | 設定変更の効果確認（次セッション）を待たずに別の変更を重ねた | settings変更は「1変更ずつ、次セッションで確認」というルールを明示化 |
| `docker-compose.yml` が最終的に Write で全書き直しになった | 複数の Edit で差分が積み重なり、ファイル全体の整合性確認が困難になった | 構成ファイルは変更前に全体を Read してから Edit する |
| Blade 4ファイルのインライン JS を一括抽出した | `no-inline-js.md` ルール追加後の既存違反の一括修正 | ルール追加時に同時に既存違反を一括修正するパターンとして記録 |
