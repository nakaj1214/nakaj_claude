## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `excel_range_select_copy.js` の outline 実装が3回アプローチ変更（box-shadow on td → input に適用 → ::after 疑似要素） | input 要素が td を覆っており、td の box-shadow が input の背後に隠れる問題を後から発見した | 編集前に `verify-before-fix` スキルで DOM 構造を DOM キャプチャで確認するフローを義務化 |
| `docker-compose.yml` の `PMA_HOSTS` を設定してすぐ `PMA_HOST` に戻す | phpMyAdmin が `PMA_HOSTS`（複数）をサポートしていないことを後から確認 | Docker イメージのドキュメント確認ステップを設ける |
| `plan.md` が12回のCodexレビューを経てようやく APPROVED（最後のセッション） | 影響範囲・非対象・既存コード精読の記述が不足していたため、毎回 CHANGES_REQUIRED が返った | `create-plan` の proposal 品質ゲート（`proposal-quality-gate` スキル）に「影響範囲の明記」「既存コード参照箇所の明記」チェック項目を追加 |
| `daily_items` パネルの配置が3回変わった（order.blade内 → order_main_area → home.blade） | 実際にレンダリングしてみないと flex レイアウトの配置が確定できない | UI コンポーネントの配置変更は小さな proof-of-concept（スタイルのみ）を先に作り、位置確定後に Blade 構造を変更する |
| `settings.json` の `disabledMcpjsonServers` を複数回変更 | キー名・効果の確認に MEMORY.md への記録よりも試行錯誤が先行した | MEMORY.md に「設定変更後は次セッションで反映確認」が記載済みだが、同セッション内では確認できないため、設定変更後に `次のセッションで確認が必要` コメントを計画に追記する習慣をつける |
