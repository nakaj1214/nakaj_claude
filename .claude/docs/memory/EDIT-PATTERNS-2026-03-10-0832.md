## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `::after` 疑似要素アプローチに至るまで2回の失敗 | UI 実装前のブラウザ描画モデル（z-index スタック）の把握が不十分 | `verify-before-fix` スキルで「DOM 構造の証拠収集」を先に行う |
| `proposal.md` を7回上書き | 要件の粒度が荒く、実装中に細部が判明するたびに書き直している | proposal 作成時に「Before/After スクリーンショット」の添付を必須化する |
| `settings.json` の matcher を往復変更 | Claude Code フックの動作検証を実機で確認できていない | 変更後すぐ `hook-debug` スキルで動作確認するフローを習慣化する |
| JS+CSS のペア変更が毎回発生 | JS 側でクラス名を動的生成し、CSS 側の定義と同期する構造になっていない | `js-css-sync-check` スキル（今セッションで導入済み）を PR 前に実行する |
