## Key Decisions

| Decision | Reason |
|----------|--------|
| `claude_p.py` で `stdin` 経由でプロンプト渡し | Windowsのコマンドライン長上限（約32KB）を超える大きなトランスクリプトを渡すため |
| `fcntl` → `msvcrt` フォールバック | Windows環境では `fcntl` が存在しない。WSLは `else` ブランチで従来通り `fcntl` を使用 |
| hooks を slack/quality/session/ に分類 | フックの種類が増えて管理しにくくなったため |
| `stop-notify.py` で `hook_event_name` チェック | Stop hookのstdinに `stop_reason` フィールドは存在しない（実際のフィールド: `hook_event_name`, `stop_hook_active`, `last_assistant_message`） |
| SubagentStop のフィルタ | settings.json の Stop hook は `SubagentStop` には反応しないが、念のためスクリプト内でも `hook_event_name == "Stop"` をチェック |

## Issues & Solutions

| 問題 | 解決策 |
|------|--------|
| git mv後にhookがパスエラー | 即座にsettings.jsonを新パスに更新 |
| `import fcntl` がWindowsで失敗 | `sys.platform == "win32"` で分岐し `msvcrt` にフォールバック |
| `claude -p` がネスト起動エラー | `env` から `CLAUDECODE` を削除してサブプロセス実行 |
| `claude -p` がコマンドライン長超過でタイムアウト | プロンプトを引数ではなく `stdin` 経由で渡すよう変更 |
| `claude.exe` がPATHに未登録 | `shutil.which` + よく使われるパスのフォールバック検索で自動解決 |
