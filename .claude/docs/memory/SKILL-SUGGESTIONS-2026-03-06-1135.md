# スキル候補レポート — 2026-03-06 11:35

## 検出されたパターン

- `.claude` と `.claude_remote` のディレクトリ比較を複数の `diff` / `find` コマンドで手動実行
- ファイル差分を「改行コードのみ」「実質的な内容変更」「プロジェクト固有（除外）」の3カテゴリに分類する作業を繰り返し実行
- 生の diff 出力（数万行）をそのまま出力 → ユーザーに「要約してほしい」と差し戻される
- 手動作業後に「抜けがないか確認して」という検証依頼
- 方針確認（上書きOKか？）を途中で挟まずに実行しようとして、ユーザーにキャンセルされる

## 観察事項

- **方針確認のタイミング**: 大量ファイルの上書き前に確認を入れようとしたが、ユーザーが「差分内容を先に見たい」と要求した。「上書き実行前に差分確認→方針合意→実行」という3ステップを先に提示する習慣が必要（ルール追加で対応可能）
- **除外ファイルリストの外部化**: 今回は毎回コード内でハードコードされた除外対象（`settings.json`, `rules/language.md` 等）を管理していた。`.claude/docs/sync-exclusions.md` などに恒久化すると次回同期が楽になる（低優先度のドキュメント整備タスク）
- **改行コード差異の根本解決**: `.editorconfig` または `git attributes` でCRLF/LFを統一すれば、毎回の「改行のみ差分」260ファイルが発生しなくなる（一度だけ対処すれば繰り返さないタスク）
