## Key Decisions

| Decision | Reason |
|----------|--------|
| `showPicker()` を `setTimeout` なしで user gesture 内から呼ぶ | DevTools閉じた状態で setTimeout 内の showPicker が "requires user gesture" で失敗するため |
| span非表示禁止 + input透明オーバーレイ方式 | spanを隠すとネイティブ date input 形式（2026/02/22）がユーザーに見えるため |
| `.col_deadline { position: relative }` + JS側でも `$td.css('position', 'relative')` | CSS specificity問題でtdがoffset parentとして機能しない場合があるため |
| `change` イベントでDB保存、`blur` はUI復元のみ | blur は「picker外クリック」でも発火するが change は「日付選択確定時のみ」発火 |
| 1課題ずつ計画書を作成して `/create-plan` → 実装 | 修正が毎回通らない問題への対策（ユーザー要望） |
| スクロール修正: `table().container()` API を使用 | `$('#stock').closest('.dataTables_scrollBody')` セレクタが found=0（DOMトラバーサル失敗）のため |

## Issues & Solutions

**課題7（要求納期）の修正が4回失敗した原因:**
- 推測ベースで修正を重ねていた
- `trigger('focus')` が click-through を起こす根本原因を DevTools 開閉で挙動差が出るまで特定できなかった
- **解決**: `verification.md` にデバッグログを記録してもらい実際の発火順序を特定

**スクロール位置保存の根本原因（今まで5回失敗）:**
- `$('#stock').closest('.dataTables_wrapper').find('.dataTables_scrollBody')` が `found = 0`
- `stock_table_ui.init()` が呼ばれる時点でDOMに `.dataTables_scrollBody` が存在しない
- DataTables が生成する要素を jQuery セレクタで追いかけるアプローチが間違い
- **対策**: `datatable.table().container()` から直接スクローラーを取得する

**ファイルパーミッション:**
- `ManagerConfirmTest.php` が root:root 所有で書き込み不可
- `docker compose exec laravel chown 1000:1000 ...` で解決
