## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `proposal.md` が同一セッションで3〜4回書き直される | 実装中に新しい問題が発覚し要件が変わる / 調査と実装を混在させている | `plan-rewrite-alert` フックを本番適用し、3回で強制停止・再整理を促す |
| `excel_range_select_copy.js` のoutline実装が3回アプローチ変更 | `td` 内部に input が全体を覆う特殊構造の把握が不十分 | 修正前に DOM構造を `verify-before-fix` で事前確認する習慣を徹底する |
| Codex 実行を `codex exec`（Bash）→ MCP 方式に移行後、全スキル・MEMORY を更新する連鎖作業が発生 | 外部ツール依存部分の変更は波及範囲が大きい | 外部ツール実行方式変更時は `docs/` に移行ノートを先に書き、影響ファイルを列挙してから一括変更する |
| daily_items パネルの配置を4回変更 | CSS flex レイアウトのコンテナ境界の設計が曖昧だった | レイアウト変更前に `antigravity` でスクリーンショット撮影し DOM 構造を把握してから変更する |
| JS + CSS のペア変更が繰り返し発生 | スタイル定義がJS内インラインCSS文字列と外部CSSに分散している | `js-css-sync-check` スキルを実装後フックに組み込み、JS編集後に自動でCSS同期チェックを促す |
