## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `excel_range_select_copy.js` に4種類のスタイル戦略を試行。`tasks/fix-attempts.md` が生成された | `input` 要素が `td` を完全に覆う構造が事前に把握できておらず、実装後に初めて問題に気づく | 修正前に `antigravity` でDOMスクリーンショットを取得し `td > input` の重なり構造を確認する手順を `verify-before-fix` に追加 |
| `settings.json` の `disabledMcpjsonServers` と `matcher` が同セッション内に4回変更 | 設定変更→テスト→不具合→再変更のサイクルが短く、設定変更の効果確認前に次の変更をしている | 設定変更後は Claudeを再起動して効果確認するまで追加変更しない旨を MEMORY.md に記録 |
| `proposal.md` が Write で全文上書きされ、REQ-003 の内容が一時的に「行選択」に誤置換された | Edit ではなく Write を使っているため既存要件との整合確認が抜ける | `docs/implement/proposal.md` への Write は `create-proposal` スキル経由のみとし、直接 Write を Hook で警告する |
| `doc/implement/plan.md` が4回書き直された。毎回「在庫テーブル」→「スタイル修正」→「outline修正」→「UI改善」と目的が変化 | 実装中に要件が変化しているが、proposal と plan を毎回白紙から書き直すため前の計画との差異が追えない | plan に変更履歴セクションを設け、propose-one スキルで「1要件1サイクル」を強制する |
