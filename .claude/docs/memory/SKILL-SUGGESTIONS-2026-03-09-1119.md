# スキル候補レポート — 2026-03-09 11:19

## 検出されたパターン

- **MCP サーバーの無効化**が2回発生（serena → playwright）。毎回手動で `settings.json` を直接編集し、1回目は誤ったキー名（`disabledMcpServers`）を使って失敗した
- **settings.json の正確性検証**を求める操作が1回。スキーマに存在しないツール名や無効なフック設定を手動で調査・修正した
- **サブエージェントの調査結果を検証なしで採用** → 設定が効かず再修正が必要になる失敗パターンが2回（disabledMcpServers の誤キー名、Notification フックの誤 matcher）
- **Docker DB 接続トラブル**（`Unknown database`、phpMyAdmin から複数DB参照）の診断・修正が複数ステップにわたって繰り返された
- **lessons.md / MEMORY.md への記録**が修正のたびに手動で行われた

## スキル候補

| スキル名 | トリガー条件 | 内容 | 優先度 |
|---------|------------|------|--------|
| `disable-mcp` | 「〇〇を無効化して」「MCPを止めて」「〇〇が毎回起動してうっとうしい」 | 正しいキー `disabledMcpjsonServers` を使って `settings.json` に追記。現在の無効化リストを確認してから追加し、次セッションで反映される旨を案内する | 高 |
| `validate-settings` | 「settings.jsonを確認して」「古くないか調べて」「設定が正しいか」 | 公式スキーマと照合して (1) 無効なツール名、(2) 誤ったフックイベント名、(3) ハードコードされたシークレット を検出・報告する | 高 |
| `docker-db-diagnose` | 「DBがコンテナと紐ついてない」「Unknown database」「接続できない」 | コンテナ内で `SHOW DATABASES` → `.env` の `DB_HOST` → マウントパスを順に確認し、不一致の原因と修正案を提示する | 中 |
