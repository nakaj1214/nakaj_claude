## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `excel_range_select_copy.js` の outline 問題に3回同じアプローチを試みた | input がセルを覆う DOM 構造を把握せずに `td` へ直接 CSS を適用しようとした | 修正前に `antigravity` でスクリーンショット取得し DOM 構造を確認（verify-before-fix の適用） |
| `proposal.md` / `plan.md` が実装中に書き直された | 要件が実装を始めてから明確になるパターン（discovery-driven development） | 要件が曖昧な段階では `brainstorming` スキルで先に整理し、実装中の要件変更は次の proposal に積み上げる |
| `order.blade.php` のパネル配置が5回移動した | Blade + CSS のレイアウト設計を頭の中で確認せずに試行錯誤した | 新しい UI 要素を追加する前に `writing-plans` スキルで DOM 設計を先にドキュメント化する |
| `settings.json` の設定値が誤って書かれ即修正された | 設定キー名をサブエージェント調査結果の鵜呑みにした（MEMORY.md に教訓あり） | settings.json 変更前に現在値を Read して確認する |
