## Issues & Solutions
- **Codex が Slack 承認で止まる問題**: `codex exec ... 2>/dev/null` のリダイレクトが `slack_approval.py` の `REDIRECT_PATTERN` に検出され require 判定になる。**解決**: リダイレクトを省いて `codex exec --sandbox workspace-write --full-auto "..."` で直接実行
- **Serena MCP のタイムアウト**: `find_symbol` / `write_memory` でタイムアウトが発生。Serena ツールが不安定な場合は Read/Grep/Edit の組み合わせで代替可能
- **サブエージェント（Task ツール）の起動拒否**: ユーザーが Task ツール起動を拒否。Codex は直接 Bash で実行することで解決

## Gotchas & Notes
- Codex のデフォルトモデルは `~/.codex/config.toml` の `gpt-5.4`（モデル指定不要、`-m o4-mini` は ChatGPT アカウントで使用不可）
- `excel_range_select_copy.js` の選択色は `injectStyles()` の `STYLE_ID` 付き `<style>` タグで注入されている。CSS 上書きではなく `style.textContent` を直接変更すること
- スクロール位置は `table_ui.js` の `STORAGE_KEY_SCROLL = 'stock_scroll_top'` で localStorage に保存/復元。仮想スクロール実装時は scrollTop の取得元（`$('.table-scroll-area').scrollTop()` 等）に注意
- `stock_table_ui.restore_scroll()` は `stock-table-rendered` イベント受信後に呼ばれる。仮想スクロール実装後もこのイベントを発火させること
