### 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| セル選択 outline を `td` の `box-shadow` で実装したが input が邪魔になった | input がセル全体を占有しており `box-shadow` が隠れる | `::after` 疑似要素オーバーレイが正解。次回から input 含む td のスタイル変更には先に DOM 構造を確認 |
| proposal.md/plan.md を都度書き直しているが、Codex レビューを通過後も実装中に要件が変わる | 実装前に UI 動作の全パターンを洗い出せていない | 計画段階で「No列クリック/範囲選択/列選択/行選択の相互作用」をチェックリスト化しておく |
| `settings.json` の PreToolUse matcher 変更→即リバート | 変更の副作用（Bash フックが無効化される）に気づいてのリバート | フック設定変更時は影響する hook イベントを事前にリストアップする |
| `docker-compose.yml` を段階修正後に Write で完全書き直し | 個別 Edit の積み上げでは構造的な問題（named volume vs bind mount）に対応しきれない | 構造変更が必要な場合は早めに Write を選択する |
