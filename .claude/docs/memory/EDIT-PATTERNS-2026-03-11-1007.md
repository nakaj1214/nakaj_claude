## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| フロントエンドUIバグ修正で同じファイルを40回編集 | `verify-before-fix` を使わずにブラウザ確認なしで実装を開始した | UIバグは必ず antigravity でスクリーンショット取得・DOM確認後に実装する |
| proposal.md を6回書き直した | 要件定義段階でブラウザの実際の動作を確認せずに仕様を決めた | proposal作成前に「現状動作」の証拠（スクリーンショット）を取得してから要件を書く |
| settings.json の `disabledMcpjsonServers` キー名を試行錯誤で発見 | MCPサーバー無効化の正しいキー名を知らなかった | MEMORY.md に記録済み（次セッション以降は即座に正しいキーを使用可能） |
| `overlay.js` の `select_all_text_input` が根本原因だったのを2セッション後に発見 | インラインスクリプトが多く、副作用のある関数の呼び出し元を追いにくかった | 今回のBladeインラインJS外部化でこの問題は軽減される見込み |
| `home.blade.php` のインラインJS抽出が失敗してロールバック | `window.storage_scope` の依存関係を事前確認せずにリファクタリングした | 外部JS化の前に `Grep` で参照元を確認してから実施する |
