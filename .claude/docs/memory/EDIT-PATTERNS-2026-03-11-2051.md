## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `settings.json` matcher を変更しては即戻す（2サイクル） | 変更前に動作仕様を確認していない | 変更前に現在のフック動作を `approval_skip_patterns.txt` と照合する手順を rules に追記 |
| `excel_range_select_copy.js` のアプローチが3回変わり 30回以上の編集が発生 | DOM 構造（`input` がセル全体を覆う）を確認せずに実装を開始した | `verify-before-fix` スキルを CSS/JS 修正にも適用。UI 変更前に DevTools でDOM 構造を確認することを義務化 |
| `home.blade.php` の `<script>` 置換→撤回 | `index_url` が JS ファイルから参照されることを把握していなかった | no-inline-js ルールに「例外ファイル一覧」セクションを追加し、移行前に参照箇所を Grep で確認する手順を明記 |
| `plan.md` の Write が 14回以上発生 | proposal.md の要件が固まる前に計画作成を開始 | `create-plan` スキルの冒頭に「proposal.md のすべての要件に Before/After が記載されているか確認」チェックを追加 |
| `docker-compose.yml` の PMA_HOSTS 設定を試行→撤回 | phpMyAdmin の環境変数仕様を確認せずに変更 | Infrastructure 変更前に公式ドキュメントまたはコンテナログで動作を確認するルールを `docker-dev` スキルに追記 |
