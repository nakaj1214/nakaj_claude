## 学習事項

| 観察 | 原因推定 | 改善アクション |
|------|---------|---------------|
| `excel_range_select_copy.js` の outline 表示に3回試行が必要だった | `input` が `td` を覆っているという DOM 構造の把握が遅れた。最初に `antigravity` でスクリーンショットを撮っていれば1回で特定できた可能性がある | UI 変更前に `verify-before-fix` スキルを必ず実行する |
| `plan.md` が12回 CHANGES_REQUIRED になった | 計画書の粒度・具体性不足。特にリスク・非対象・影響ファイルの列挙が後から追加されていった | `proposal-quality-gate` を `create-plan` の前に必ず通す |
| `settings.json` の matcher を4回変更した | 変更後にセッションを再起動して動作確認するまでに時間がかかった | hooks の変更後は `hook-debug` スキルで即確認する習慣をつける |
| `daily_items` パネルが4回移動した | レイアウト設計（どの HTML 要素の子にするか）を実装前に固めていなかった | CSS レイアウト変更を伴う機能は `brainstorming` スキルで構造を先に決定する |
| Codex レビューが毎回 `CHANGES_REQUIRED` で1点ずつ修正した | 修正が1指摘1対応になっており、全指摘を読んでから一括対応していなかった | `review.md` を読んで全指摘をリストアップしてから plan.md を修正する |
