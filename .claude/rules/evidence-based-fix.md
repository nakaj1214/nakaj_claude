# Evidence-Based Fix（証拠に基づく修正）

**推測で修正してはならない。証拠を集めてから修正すること。全言語・全レイヤーに適用。**

詳細な5フェーズプロセス・検証手段・停止条件は `/verify-before-fix` スキルを参照。
このルールはスキル起動前の **最低限のゲート** として機能する。

---

## 修正前の3原則

### 1. 現象を事実ベースで記述する

```
❌ 「たぶん○○が原因だろう」
✅ 「ユーザー報告: ○○が発生する。原因は未特定。」
```

### 2. 仮説には必ず検証方法をセットで提示する

証拠なき仮説で修正を始めてはならない。

```
❌ 「おそらく○○が原因なので△△を調整する」
✅ 「○○が原因かどうか確認するため、以下を検証します/してください: ...」
```

### 3. コードを読んだだけで断定しない

Claude がコードを読んで得られるのは **静的な情報** のみ。実行時の動作は確認できていない。

```
❌ 「コードを読んだところ、このイベントハンドラーが原因です」
✅ 「コードからは X が原因の可能性があります。[antigravity/ログ/テスト] で確認します。」
```

---

## 停止条件

以下のパターンに陥ったら **即座に止まり**、`/verify-before-fix` スキルを起動する:

- 同じ問題に対する2回目の修正を試みている
- CSS 値・setTimeout・!important・try-catch を繰り返し調整している
- `.env` / config の値を何度も変えている
