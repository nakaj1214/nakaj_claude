# エージェントオーケストレーション

出典: [everything-claude-code](https://github.com/affaan-m/everything-claude-code) `rules/common/agents.md`

## 利用可能なエージェント

`.claude/agents/` 内の特化エージェント:

| エージェント | 主な用途 |
|------------|---------|
| planner | 複雑な機能の実装計画、リファクタリング |
| architect | システム設計、アーキテクチャ判断 |
| tdd-guide | テスト駆動開発の実践 |
| code-reviewer | コードレビュー、品質評価 |
| security-reviewer | セキュリティ分析 |
| build-error-resolver | ビルドエラーの解決 |
| e2e-runner | 重要なユーザーフローの E2E テスト |
| refactor-cleaner | デッドコードの削除、メンテナンス |
| doc-updater | ドキュメントの更新 |

## エージェントの自動起動

以下のシナリオでは指示を待たずにエージェントを起動する:

- **複雑な機能リクエスト** → `planner` エージェントを使用
- **最近変更されたコード** → `code-reviewer` エージェントを使用
- **バグ修正または新機能** → `tdd-guide` エージェントを使用
- **設計上の判断が必要** → `architect` エージェントを使用

## 実行戦略

### 並列実行

**独立した操作には常に並列 Task 実行を使用する。**

```
# 悪い例: 順次実行
Task: セキュリティ分析 → 完了を待つ
Task: パフォーマンスレビュー → 完了を待つ
Task: 型チェック → 完了を待つ

# 良い例: 並列実行
Task: セキュリティ分析 ┐
Task: パフォーマンスレビュー ├→ 全結果を収集
Task: 型チェック ┘
```

### マルチパースペクティブ分析

複雑な問題には分割ロールのサブエージェントを使用:

| ロール | 目的 |
|-------|------|
| 事実確認レビュアー | コードの正確性を検証 |
| シニアエンジニア | アーキテクチャとベストプラクティスを評価 |
| セキュリティ専門家 | 脆弱性を検出 |
| 一貫性レビュアー | コードベース全体の整合性を確認 |
| 冗長性チェッカー | 重複コードやパターンを検出 |
