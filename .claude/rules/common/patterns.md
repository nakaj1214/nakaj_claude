# 共通パターン

出典: [everything-claude-code](https://github.com/affaan-m/everything-claude-code) `rules/common/patterns.md`

## スケルトンプロジェクト

新しい機能を実装する際:

1. 実績のあるスケルトンプロジェクトを検索する
2. 並列エージェントを使って候補を評価する:
   - セキュリティ評価
   - 拡張性分析
   - 関連性スコアリング
   - 実装計画
3. 最適な候補をベースとしてクローンする
4. 実績のある構造の中でイテレーションする

## デザインパターン

### リポジトリパターン

一貫したインターフェースの背後にデータアクセスをカプセル化する:

- 標準操作を定義: `findAll`, `findById`, `create`, `update`, `delete`
- 具象実装がストレージの詳細を処理（データベース、API、ファイル等）
- ビジネスロジックはストレージ機構ではなく抽象インターフェースに依存する
- データソースの容易な交換を可能にし、モックによるテストを簡素化する

### API レスポンス形式

すべての API レスポンスに一貫したエンベロープを使用する:

```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "meta": {
    "total": 100,
    "page": 1,
    "limit": 20
  }
}
```

- `success` / `status` インジケータを含める
- データペイロード（エラー時は null）を含める
- エラーメッセージフィールド（成功時は null）を含める
- ページネーションレスポンスにはメタデータ（`total`, `page`, `limit`）を含める
