# 単一課題の計画実行 — 詳細手順

複数課題が prompt.md に並んでいるとき、1つだけ選んで確実に計画を立てる。
複数課題を同時進行させると混乱が生じるため、このスキルで1課題=1計画を強制する。

---

## ワークフロー

### Step 1: prompt.md を読む

```
docs/implement/prompt.md を読む
```

課題が複数ある場合はリストアップして、**どれを実施するかユーザーに確認する**（自動選択しない）。

課題が1つのみの場合はそのまま進む。

### Step 2: 選択した課題のみで proposal を生成

`docs/implement/proposal.md` に以下の形式で出力:

```markdown
# Proposal: [課題名]

## 対象課題
[prompt.md から抜粋した該当箇所]

## 問題の整理
[何が問題か、背景]

## 解決方針
[どのように解決するか]

## 実装スコープ
- [ ] [具体的な変更点1]
- [ ] [具体的な変更点2]

## 対象外（今回は実施しない）
- [他の課題や追加改善]

## 完了条件
- [何ができたら完了か]
```

### Step 3: plan を生成

`docs/implement/plan.md` に2〜5分単位のステップで出力:

```markdown
# Plan: [課題名]

## ステップ

### Step 1: [作業名]
- 対象ファイル: [ファイルパス]
- 作業内容: [具体的な操作]
- 完了確認: [どうなれば完了か]

### Step 2: [作業名]
...

## 注意点
- [実装時の注意事項]
```

### Step 4: 停止してユーザーに確認

plan.md 生成後に **停止する**。実装は開始しない。

```
plan.md を作成しました。確認後に実装を開始してください。
```

---

## ルール

- 選択した課題以外の変更を plan に含めない
- 「ついでに〜も直す」は禁止
- prompt.md の他の課題は `docs/implement/prompt.md` に残したまま手をつけない
- plan 生成後は自動実行しない（`/implement-plans` で明示的に実行）
