ビルドエラーと型エラーを、最小限の段階的な変更で体系的に解決する。

## ワークフロー

### 1. ビルドシステムの検出
マニフェストファイルからプロジェクトの種類を特定する:
- `package.json` → `npm run build` / `tsc --noEmit`
- `Cargo.toml` → `cargo build`
- `pom.xml` → `mvn compile`
- `go.mod` → `go build ./...`
- `pyproject.toml` → `python -m py_compile`

### 2. エラーの収集と整理
```bash
# Run build and capture all errors
npm run build 2>&1 | tee build-errors.txt
```

エラーを以下の順序でグループ化する:
1. インポート/依存関係の問題（最初に修正）
2. 型エラー
3. ロジックレベルのエラー

### 3. 反復的な解決ループ
各エラーに対して:
1. 影響を受けるコードセクションを読む
2. 根本原因を特定する
3. 可能な限り最小の修正を適用する
4. リビルドして改善を確認する

### 4. エスカレーションチェックポイント
以下の場合はユーザーに確認を取る:
- 修正が追加のエラーを生成する場合
- 同じエラーが3回の試行後も解消しない場合
- 変更に構造的な再設計が必要な場合
- 不足しているパッケージがインストールできない場合

### 5. 復旧テクニック

| 問題 | 解決策 |
|---------|---------|
| パッケージの不足 | `npm install <pkg>` または `pip install <pkg>` |
| 型の非互換性 | 型アサーションの追加または実際の型を修正 |
| 循環インポート | 共有コードを新しいモジュールに抽出 |
| バージョンの競合 | バージョンを固定またはロックファイルを更新 |
| 設定エラー | tsconfig/webpack/vite の設定を確認 |

**原則:** 一度に1つのエラーのみ対処する。無関係なコードをリファクタリングしない。
