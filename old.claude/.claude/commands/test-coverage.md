テストカバレッジを分析し、80%以上のカバレッジ閾値に到達するために不足しているテストを生成します。

## ステップ 1: フレームワークの検出とカバレッジの実行

```bash
# フレームワーク検出
ls jest.config.* vitest.config.* pytest.ini 2>/dev/null

# カバレッジ実行
npx jest --coverage                          # Jest
npx vitest run --coverage                    # Vitest
pytest --cov=. --cov-report=term-missing     # pytest
go test -coverprofile=coverage.out ./...     # Go
```

## ステップ 2: ギャップの特定

カバレッジレポートから以下を検出:
- カバレッジ不足のファイル（80%未満）
- テストされていない関数やメソッド
- ブランチカバレッジの欠落（if/else、switch、エラーパス）
- 分母を膨らませているデッドコード

## ステップ 3: テスト生成の優先順位

各ギャップに対して、以下の順序でテストを作成:
1. **正常系** — 有効な入力、期待される出力
2. **異常系** — 無効な入力、エラー返却
3. **境界値** — 空、null、ゼロ、最大値
4. **ブランチカバレッジ** — すべての条件分岐

## テスト品質基準

- 各テストは独立（共有される可変状態なし）
- 説明的な命名: `test_create_user_with_duplicate_email_returns_409`
- ファイルの場所とインポートについて既存のプロジェクトテスト規約に従う
- 外部サービスをモック — テストは実際のネットワーク呼び出しを行わない

## 優先ターゲット

以下に焦点を当てる:
1. 高複雑度関数（循環的複雑度 > 5）
2. エラーハンドリングパス
3. ユーティリティとヘルパー関数
4. API エンドポイントとコントローラー
5. エッジケース: null、空のコレクション、数値の境界

