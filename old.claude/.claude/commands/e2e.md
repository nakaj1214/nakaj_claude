重要なユーザージャーニーのエンドツーエンドテストを生成・実行する。e2e-runner エージェントに委譲する。

## 使用するタイミング

複数ステップのユーザーフローのテストに使用する:
- 認証（ログイン、ログアウト、ユーザー登録）
- コアビジネスワークフロー（チェックアウト、決済、取引）
- ユーザーデータの CRUD 操作
- サービス間統合

**金融フローはテストネット/ステージングのみで実行すること。本番環境では絶対に実行しない。**

## 実行内容

1. **ジャーニー分析** -- コードベース内の重要なユーザーパスを特定する
2. **テスト生成** -- Page Object Model を使用して Playwright テストを作成する
3. **マルチブラウザ実行** -- Chromium、Firefox、WebKit
4. **アーティファクト取得** -- 失敗時のスクリーンショット、動画、トレース
5. **フレーキーテスト検出** -- 不安定なテストを特定して隔離する

## アーティファクト出力

すべての実行で生成されるもの:
- HTML テストレポート
- JUnit XML 結果

失敗時に追加で生成されるもの:
- 失敗時点のスクリーンショット
- テストセッションの動画記録
- Playwright トレース（ステップごとのリプレイ）
- ネットワークおよびコンソールログ

## 品質目標

| メトリクス | 目標 |
|--------|--------|
| 重要ジャーニー | 100% パス |
| 全体パス率 | >95% |
| フレーキーテスト率 | <5% |
| スイート実行時間 | <10分 |

## フレーキーテストの対処

断続的な失敗が発生した場合、以下の推奨事項がトリガーされる:
- 明示的な待機条件を追加する
- 要素の可視性タイムアウトを増やす
- 非同期操作における競合状態をチェックする
- ネットワーク依存のステップにリトライロジックを追加する
