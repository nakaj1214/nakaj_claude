# ライブラリ調査

$ARGUMENTS を調査して `.claude/docs/libraries/` にドキュメントを作成する。

## 調査項目

### ウェブ検索で必ず確認すること

- 公式ドキュメント
- GitHub README
- PyPI / npmページ
- 最新リリースノート

### ドキュメント化する内容

1. **基本情報**
   - 正式名称、バージョン、ライセンス
   - 公式URL
   - インストールコマンド

2. **主要機能**
   - メイン機能
   - 基本的な使い方（コード例）

3. **制約・注意事項**
   - 既知の制限
   - 他のライブラリとの競合
   - パフォーマンス特性
   - 非同期/同期の考慮点

4. **このプロジェクトでの使用パターン**
   - 推奨する使い方
   - 避けるべきパターン

5. **トラブルシューティング**
   - よくあるエラーと解決策

## 出力先

`.claude/docs/libraries/$ARGUMENTS.md`

## ドキュメントテンプレート

```markdown
# {ライブラリ名}

## 概要

- **バージョン**: {version}
- **ライセンス**: {license}
- **公式URL**: {url}
- **インストール**: `{インストールコマンド}`

## 主要機能

{メイン機能の説明}

## 基本的な使い方

```python
{コード例}
```

## 制約・注意事項

- {制限1}
- {制限2}

## 推奨パターン

### 良い例

```python
{良いパターン}
```

### 避けるべき例

```python
{アンチパターン}
```

## トラブルシューティング

### {エラーメッセージ}

**原因**: {原因}
**解決策**: {解決策}

## 参照

- [公式ドキュメント]({url})
- [GitHub]({url})
```

## 既存ドキュメントの更新 (from update-lib-docs)

既にドキュメント化済みのライブラリを更新する場合のワークフロー。

### バッチ更新手順

1. `ls .claude/docs/libraries/` で既存ドキュメントをリストアップ
2. 各ライブラリの最新情報をウェブ検索
3. 変更があったドキュメントを更新
4. コードへの影響を確認

### 確認項目

| カテゴリ | 確認内容 |
|--------|---------|
| セキュリティ | CVE、セキュリティパッチ |
| 破壊的変更 | API変更、削除された機能 |
| 非推奨 | 削除予定のAPI |
| パフォーマンス | 最適化の改善 |
| 新機能 | 有用な追加機能 |

### 更新後チェック

- 非推奨APIを使用していないか？
- 破壊的変更の影響はないか？
- プロジェクトの依存関係を更新する必要があるか？
