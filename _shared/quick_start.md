# Claude Code クイックスタートガイド

Claudeを最大限に活用するための最速スタートガイドです。

## 🚀 5分で始める

### ステップ1: 基本原則を理解する
Claudeと作業する際の3つの黄金律:

1. **読んでから編集** - ファイルを変更する前に必ず読む
2. **最小限の変更** - 要求されたことだけを行う
3. **具体的に依頼** - 曖昧な指示ではなく、明確な指示を出す

### ステップ2: 最初のタスク
```
# Good ✅
"src/index.tsを読んで、main関数に何があるか教えてください"

# Bad ❌
"コードを見て"
```

### ステップ3: 効果的な依頼方法
```
# 機能追加の場合
"[ファイル名]の[場所]に[機能]を追加してください"

# バグ修正の場合
"[ファイル名]で[問題]が発生しています。修正してください"

# 調査の場合
"[機能名]はどこに実装されていますか？"
```

## 💡 頻繁に使う操作

### コードベース探索
```
「認証機能はどこに実装されていますか？」
→ Exploreエージェントが自動的に検索
```

### ファイル編集
```
「login.tsのhandleLogin関数にエラーハンドリングを追加してください」
→ 自動的にファイルを読み、編集を提案
```

### Git操作
```
「変更をコミットしてください」
→ git status確認 → 変更確認 → コミット作成
```

### テスト実行
```
「テストを実行して、失敗があれば修正してください」
→ テスト実行 → エラー確認 → 修正 → 再テスト
```

## ⚡ パワーユーザーのヒント

### 1. 並列実行
複数の独立したタスクを同時に実行:
```
「テストを実行しながら、ビルドも実行してください」
```

### 2. エージェント活用
複雑な探索にはExploreエージェントを使用:
```
「エラーハンドリングがどのように実装されているか、包括的に調査してください」
```

### 3. TODO管理
複雑なタスクは自動的にTODOリストで管理される:
```
「ユーザー認証機能を実装してください」
→ 自動的にステップが分割され、進捗が追跡される
```

### 4. プランモード
大きな変更の前に計画を立てる:
```
「ダークモード機能を追加したい」
→ Claudeが実装計画を作成し、承認を求める
```

## 🛡️ よくある失敗を避ける

### ❌ 避けるべきこと

1. **曖昧な指示**
```
Bad: "コードを改善して"
Good: "未使用の変数を削除してください"
```

2. **一度に多すぎる要求**
```
Bad: "ログイン、登録、パスワードリセット、2FA を実装して"
Good: まずログインから、次に登録、というように段階的に
```

3. **読まずに編集を依頼**
```
Bad: "auth.tsのlogin関数を修正して"
（Claudeがファイルを読んでいない場合）
Good: "auth.tsを読んで、login関数の問題を教えてください"
```


## 💬 コミュニケーションのコツ

### Claudeとの対話
- **明確に**: 何を、どこで、どのように
- **具体的に**: ファイル名、関数名、行番号
- **段階的に**: 大きなタスクは小さく分割
- **確認しながら**: 不明点があれば質問する

### フィードバックの提供
```
Good: "いいですね。ただし、エラーメッセージは日本語にしてください"
Good: "ほぼ正しいですが、validation関数の戻り値の型が間違っています"
```
