# ライブラリドキュメントを更新する

`.claude/docs/libraries/` のドキュメントを最新情報に更新する。

## ステップ

### 1. 既存ドキュメントを確認する

現在のライブラリドキュメントをリストアップする:

```bash
ls .claude/docs/libraries/
```

### 2. 最新情報をウェブ検索する

ドキュメント化されている各ライブラリについて検索する:

- 最新バージョン
- 破壊的変更
- 非推奨となった機能
- 新機能
- セキュリティアップデート

### 3. ドキュメントを更新する

変更があった各ライブラリについて:

1. バージョン情報を更新する
2. 新機能/制約を追加する
3. 非推奨APIをマークする
4. 必要に応じてコード例を更新する
5. ファイルの先頭に更新日を記録する

### 4. コードへの影響を確認する

ドキュメント更新後に確認する:

- 非推奨APIを使用していないか？
- 破壊的変更の影響はないか？
- プロジェクトの依存関係を更新する必要があるか？

## 確認すべき主要項目

| カテゴリ | 確認内容 |
|--------|---------|
| セキュリティ | CVE、セキュリティパッチ |
| 破壊的変更 | API変更、削除された機能 |
| 非推奨 | 削除予定のAPI |
| パフォーマンス | 最適化の改善 |
| 新機能 | 有用な追加機能 |

## 更新フォーマット

ファイルの先頭に更新通知を追加する:

```markdown
# {ライブラリ名}

> **最終更新**: {日付}
> **確認バージョン**: {version}

## 最近の変更

- {変更1}
- {変更2}

---

{ドキュメントの残り部分}
```

## 報告

更新後、ユーザーに日本語で報告する:

- 更新したライブラリ
- 発見した重要な変更点
- プロジェクトに必要なアクション
