# Claude Prompt Templates

効果的なプロンプトのテンプレート集です。これらを使用することで、Claudeからより良い結果を得られます。

## コードベース探索

### パターン1: 機能の場所を探す
```
[機能名]はどこに実装されていますか？
具体的なファイルと行番号を教えてください。
```

### パターン2: アーキテクチャを理解する
```
このプロジェクトの[モジュール/システム名]のアーキテクチャを説明してください。
主要なファイル、データの流れ、依存関係を含めて教えてください。
```

### パターン3: エラーハンドリングを調査
```
[コンポーネント/モジュール名]でのエラーハンドリングはどのように実装されていますか？
エラーの種類と処理方法を教えてください。
```

- [要件3]

## 効果的なプロンプトのコツ

### ✅ Good Prompts

1. **具体的**
```
Good: "src/auth/login.tsのhandleLoginメソッドにメール検証を追加してください"
Bad: "ログイン機能を改善して"
```

2. **コンテキスト提供**
```
Good: "ユーザーがログインフォームを送信する際、無効なメールアドレスの場合はエラーを表示したい"
Bad: "バリデーション追加"
```

3. **制約を明確に**
```
Good: "既存のバリデーションライブラリ（Yup）を使用し、既存のエラー処理パターンに従ってください"
Bad: "バリデーション追加して"
```

4. **段階的な指示**
```
Good:
"1. まず認証フローを調査
 2. 次にセキュリティ問題を確認
 3. 問題があれば修正案を提示"
Bad: "認証を直して"
```

### ❌ Avoid

1. **曖昧な指示**
```
避けるべき: "コードをきれいにして"
改善版: "未使用の変数とコメントアウトされたコードを削除してください"
```

2. **複数の無関係なタスク**
```
避けるべき: "ログイン機能追加、データベース最適化、UI改善をして"
改善版: 個別のタスクに分割して順次依頼
```

3. **時間に関する質問**
```
避けるべき: "これは何分でできる？"
改善版: "このタスクに必要なステップを教えてください"
```

## 言語別のプロンプト例

### JavaScript/TypeScript
```
[関数名]をTypeScriptで実装してください。
型定義を含め、strictモードに準拠してください。
```

### Python
```
[機能名]をPythonで実装してください。
PEP 8に従い、型ヒントを含めてください。
```

### React
```
[コンポーネント名]というReactコンポーネントを作成してください。
Hooks を使用し、既存のコンポーネントパターンに従ってください。
```

### SQL
```
[目的]のためのSQLクエリを作成してください。
SQLインジェクション対策を含め、パラメータ化してください。
```

## まとめ

- **明確で具体的**なプロンプトを使用
- **コンテキスト**を提供
- **制約や要件**を明示
- **段階的なアプローチ**で複雑なタスクに取り組む
- 必要に応じて**対話的に**詳細を詰める
